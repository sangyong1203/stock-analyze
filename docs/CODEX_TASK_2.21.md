# CODEX_TASK_2.21

이전 작업 컨텍스트를 유지하고 이어서 진행하세요.

이번 작업은 **프론트 인증 상태 유지 / 보호 라우트 / 로그아웃 최소 구현**입니다.

## 작업

1. 현재 로그인 성공 후 인증 상태가 어떻게 유지되는지 확인하세요.
2. `/dashboard`, `/portfolio`, `/alerts`, `/news`, `/settings`를 로그인 필요 화면으로 정리하세요.
3. 인증 상태가 없으면 `/login`으로 이동하게 하세요.
4. 로그인 성공 후 `/dashboard` 접근이 유지되는지 확인하세요.
5. 로그아웃 버튼 또는 최소 로그아웃 동작을 구현하세요.
6. 새 테이블 / 새 마이그레이션은 만들지 마세요.
7. 실제 Gmail 발송은 하지 마세요.
8. 결과를 `docs/DEVELOPMENT_REPORT.md`에 정리하세요.

## 기준

- 기존 Google OAuth 흐름은 유지
- 과도한 권한/역할 기능 추가 금지
- 개인 MVP 기준의 최소 인증 구조만 구현
- 새 인증 라이브러리 추가는 필요할 때만
