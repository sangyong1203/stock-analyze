# 투자 분석 시스템 개발 계획서 v1.2

## 1. 프로젝트 개요

본 프로젝트는 개인 투자 기록, 뉴스 수집, 종목 관리, 거래 사유 기록, 목표가 알림, 차트 확인, 메모/태그 관리를 통합하기 위한 개인용 투자 분석 시스템이다.

본 시스템은 외부 공개 서비스가 아니라 개인 단독 사용을 전제로 한다.
1차 개발 범위는 한국 주식 중심으로 한다.

---

## 2. 개발 목적

본 시스템의 핵심 목적은 투자 판단의 근거를 기록하고, 이후 다시 분석할 수 있는 구조를 만드는 것이다.

주요 목적은 다음과 같다.

- 종목별 뉴스 수집 및 중요 뉴스 선별
- 매수/매도 기록과 사유 저장
- 거래 당시 가격과 관련 뉴스 연결
- 목표가 및 중요 뉴스 알림
- 보유종목, 관심종목, 알림종목 관리
- 투자 메모와 태그 관리
- 차트에서 매수/매도 기록 확인
- 거래 기록 기반 투자 판단 검토 기반 마련

# 7. 메뉴별 기능 정의

## 7.1 대시보드

대시보드는 투자 현황과 주요 알림을 요약해서 보여주는 화면이다.

주요 기능:

- 총 투자금 요약
- 현금 잔액 요약
- 보유 평가금액 요약
- 평가손익 요약
- 실현손익 요약
- 오늘 수집된 중요 뉴스 표시
- 알림 대상 뉴스 표시
- 목표가 접근 종목 표시
- 보유종목 주요 변동 표시

## 7.2 종목

종목 메뉴는 시스템에서 사용하는 종목 정보를 조회하고, 뉴스 매칭에 필요한 최소 정보를 보정하는 화면이다.

주요 기능:

- 종목 목록 조회
- 종목 상세 조회
- 종목 코드 확인
- 종목명 확인
- 시장 구분 확인
- 섹터/업종 확인
- 현재가 확인
- 시가총액 확인
- KODEX 200 포함 여부 확인
- KODEX 코스닥150 포함 여부 확인
- 뉴스 매칭용 별칭 확인 및 보정
- 보유 여부 확인
- 관심 여부 확인
- 알림 설정 여부 확인

종목 메뉴의 역할은 종목 기본정보 확인과 일부 보정이다.
실제 뉴스 수집 대상 조건 관리는 `수집 종목 관리` 메뉴에서 처리한다.

## 7.3 수집 종목 관리

수집 종목 관리 메뉴는 실제 뉴스 수집 대상 종목을 관리하는 화면이다.

KODEX 200과 KODEX 코스닥150 구성종목 350개는 기초 후보군으로 사용한다.
실제 수집 대상은 조건과 수동 설정을 통해 별도로 산출한다.

### 주요 기능

- KODEX 200 구성종목 목록 확인
- KODEX 코스닥150 구성종목 목록 확인
- 구성종목 파일 반영
- 구성종목 수동 갱신
- 수집 조건 설정
- 최종 수집 대상 산출
- 종목별 수집 ON/OFF
- 종목별 수동 포함
- 종목별 수동 제외
- 현재 최종 수집 대상 종목 수 확인
- 수집 제외 종목 확인
- 수집 대상 산출 사유 확인

### 수집 대상 조건

- KODEX 200 포함 여부
- KODEX 코스닥150 포함 여부
- 시가총액 기준
- 시장 구분
- 섹터
- 보유종목 여부
- 관심종목 여부
- 알림 설정 종목 여부
- 종목별 수동 포함/제외

### 기본 조건 예시

- 시가총액 1조 이상만 수집
- KODEX 200만 수집
- KODEX 코스닥150만 수집
- 보유종목은 조건과 관계없이 포함
- 관심종목은 조건과 관계없이 포함
- 사용자가 수동 제외한 종목은 조건을 만족해도 제외

### 수집 대상 우선순위

1. 수동 제외
2. 수동 포함
3. 보유종목
4. 관심종목
5. 알림 설정 종목
6. 조건 규칙

## 7.4 뉴스

뉴스 메뉴는 수집된 뉴스, 필터링 결과, GPT 요약, 관련 종목 정보를 확인하는 화면이다.

### 주요 기능

- 전체 뉴스 목록 조회
- 중요 뉴스 목록 조회
- 종목별 뉴스 조회
- 시장 뉴스 조회
- 섹터 뉴스 조회
- 매크로 뉴스 조회
- 정책 뉴스 조회
- 제외 후보 뉴스 조회
- GPT mini 요약 결과 조회
- GPT-5 재필터 결과 조회
- 뉴스 원문 URL 확인
- 관련 종목 확인
- 뉴스 당시 주가 확인
- 뉴스별 중요도 점수 확인
- 뉴스별 주가 영향 판단 확인
- 중복 보도 횟수 확인
- 보도 언론사 수 확인
- 보도 언론사 목록 확인
- 최초 보도 시간 확인
- 최종 보도 시간 확인

### 뉴스 분류

뉴스는 다음 범위로 분류한다.

- stock
- sector
- market
- macro
- policy
- noise

### 뉴스 상태

뉴스는 필터링 결과에 따라 다음 상태를 가진다.

- important_candidate
- normal_candidate
- exclude_candidate
- review_needed

### 뉴스 저장 정책

- 중복 뉴스 원문은 저장하지 않는다.
- 대표 뉴스 1건만 저장한다.
- 중복 보도는 대표 뉴스에 누적한다.

대표 뉴스에 누적 저장하는 항목:

- duplicate_count
- source_count
- sources_json
- first_published_at
- last_published_at

## 7.7 알림 관리

알림 관리 메뉴는 가격 알림, 뉴스 알림, 알림 이력을 관리하는 화면이다.

### 주요 기능

- 가격 알림 목록 조회
- 가격 알림 등록
- 가격 알림 수정
- 가격 알림 삭제
- 가격 알림 ON/OFF
- 뉴스 알림 대상 확인
- 뉴스 알림 기준 확인
- 뉴스 알림 ON/OFF
- 알림 발송 이력 조회
- 알림 발송 실패 이력 조회
- 일일 알림 한도 확인
- 시간당 알림 한도 확인

### 가격 알림 유형

- 직접 목표가
- 1달 내 고점 대비 하락률
- 1달 내 저점 대비 상승률

예시:

- 삼성전자 90,000원 도달 시 알림
- 최근 1달 고점 대비 10% 하락 시 알림
- 최근 1달 저점 대비 15% 상승 시 알림

### 뉴스 알림 기본 대상

- 보유종목
- 관심종목
- 알림 설정 종목

### 뉴스 알림 기본 조건

- importance_score 7점 이상
- 또는 duplicate_count 3회 이상
- 또는 이벤트성 중요 뉴스

### 이벤트성 중요 뉴스

- 공시
- 실적
- 수주
- 공급계약
- 유상증자
- 무상증자
- 배당
- 자사주
- 주주환원
- 증설
- CAPA 확대
- 인수합병
- 소송
- 제재
- 거래정지
- 관리종목

알림 기준의 상세 설정은 `설정` 메뉴에서 관리한다.

- 제외 후보 기준 설정

기본 중복 체크 기간:

- 속보/시장/일반 뉴스: 24시간
- 실적/공시/수주/증자 등 이벤트성 뉴스: 3일

### 알림 설정

- 뉴스 알림 사용 여부
- 가격 알림 사용 여부
- importance_score 기준 설정
- duplicate_count 기준 설정
- source_count 기준 설정
- 알림 대상 범위 설정
- 이벤트 키워드별 알림 ON/OFF
- 하루 최대 알림 수 설정
- 시간당 최대 알림 수 설정

기본값:

- importance_score 7점 이상
- duplicate_count 3회 이상
- 하루 최대 200건
- 1시간 최대 50건

# 8. 백엔드 자동 처리 기능

이 영역은 사용자가 직접 보는 메뉴가 아니라, 백엔드에서 자동 또는 수동 실행으로 처리되는 기능이다.

---

## 8.1 주가 데이터 수집

### 일봉/주봉/월봉 데이터

KRX Open API를 우선 사용한다.

수집 데이터:

- 시가
- 고가
- 저가
- 종가
- 거래량
- 거래대금
- 시가총액
- 상장주식수
- 등락률

### 시가총액

시가총액은 KRX 일별매매정보의 `MKTCAP`을 사용한다.

시가총액은 수집 종목 조건 판단에 사용한다.

### 현재가 스냅샷

네이버 증권 현재가 스냅샷을 사용한다.

저장 시점:

- 뉴스 저장 시점
- 매수 기록 시점
- 매도 기록 시점
- 알림 발생 시점
- 메모 작성 시점

네이버 현재가 방식이 불안정하면 한국투자증권 Open API를 대안으로 검토한다.

---

## 8.2 구성종목 업데이트

KODEX 200과 KODEX 코스닥150 구성종목을 업데이트한다.

업데이트 방식:

- 매월 1일 자동 갱신
- 수동 버튼 클릭 갱신
- KRX에서 다운로드한 CSV/엑셀 파일 반영

구성종목은 실제 수집 대상이 아니라 기초 후보군으로 사용한다.

---

## 8.3 수집 종목 산출

기초 후보군과 수집 조건을 기준으로 실제 뉴스 수집 대상 종목을 산출한다.

산출 기준:

- KODEX 200 포함 여부
- KODEX 코스닥150 포함 여부
- 시가총액 조건
- 시장 조건
- 섹터 조건
- 보유종목 여부
- 관심종목 여부
- 알림 설정 종목 여부
- 수동 포함 여부
- 수동 제외 여부

산출 결과:

- collect_enabled
- collect_reason
- priority
- manual_include
- manual_exclude

---

## 8.4 뉴스 수집

메인 수집 소스:

- 네이버 금융 시장 뉴스
- `https://finance.naver.com/news/news_list.naver`

수집 주기:

- 매일 07:30 ~ 23:30
- 1시간 간격
- 30분 기준 실행

수동 실행도 가능해야 한다.

종목별 뉴스 페이지는 전체 순회용이 아니라 다음 경우에만 보조적으로 사용한다.

- 보유종목
- 관심종목
- 알림 설정 종목
- 시장 뉴스에서 중요 후보로 잡힌 종목

---

## 8.5 뉴스 필터링

뉴스 제목, 요약, preview에서 종목명과 별칭을 추출한다.

처리 흐름:

1. 뉴스 수집
2. 뉴스 정규화
3. 종목명/별칭 매칭
4. KODEX 200 또는 KODEX 코스닥150 구성종목 여부 확인
5. 실제 수집 대상 조건 확인
6. 조건 통과 시 저장 후보 처리
7. 미포함 개별 종목 뉴스는 제외
8. 종목 매칭이 없어도 시장/섹터/매크로/정책 키워드가 있으면 저장 후보 처리
9. 제외 키워드가 강하게 매칭되면 제외 후보 처리

---

## 8.6 뉴스 키워드 판단

시장/섹터/매크로/정책/제외/이벤트 키워드를 기준으로 뉴스를 분류한다.

키워드 그룹:

- market
- sector
- macro
- policy
- exclude
- event

기본 키워드 프리셋을 제공하고, 설정 화면에서 추가/수정/삭제/ON-OFF/가중치 설정이 가능해야 한다.

---

## 8.7 뉴스 중복 처리

중복 뉴스 원문은 저장하지 않는다.

중복 체크 기준:

- url_hash
- title_hash
- news_group_key

중복 체크 기간:

- 속보/시장/일반 뉴스: 최근 24시간
- 실적/공시/수주/증자 등 이벤트성 중요 뉴스: 최근 3일

중복 처리 방식:

1. 새 뉴스 수집
2. url_hash 생성
3. title_hash 생성
4. news_group_key 생성
5. 최근 중복 체크 기간 안에서 기존 대표 뉴스 조회
6. 중복이면 신규 원문 저장하지 않음
7. 기존 대표 뉴스의 duplicate_count 갱신
8. 기존 대표 뉴스의 source_count 갱신
9. 기존 대표 뉴스의 sources_json 갱신
10. 기존 대표 뉴스의 last_published_at 갱신
11. importance_score 재계산
12. 중복이 아니면 신규 대표 뉴스 저장

---

## 8.8 뉴스 중요도 점수 계산

뉴스 중요도는 0~10점 기준으로 계산한다.

기본 점수:

- 공시/수주/증자/실적/배당/자사주: +5
- 증설/CAPA/신공장/인수합병: +4
- 정책/규제/세제/보조금: +4
- 기술/특허/인증/양산/임상: +3
- 기술 성과 발표: 우선 포함
- 시장/매크로 핵심 뉴스: +3
- 섹터 업황 뉴스: +2

가산점:

- duplicate_count >= 3: +2
- duplicate_count >= 5: +3
- source_count >= 3: +1
- source_count >= 5: +2
- 보유종목: +2
- 관심종목: +1
- 알림 설정 종목: +2

감점:

- 단순 특징주: -2
- 단순 장중 급등락: -2
- 행사/봉사/사진 기사: -4
- 광고성/프로모션: -5
- 미포함 개별 종목 뉴스: 제외

점수 해석:

- 0~3점: 저장 제외 또는 낮은 중요도
- 4~5점: 저장 가능, GPT 요약 제외 가능
- 6~7점: GPT 요약 후보
- 8~10점: 중요 뉴스, 알림 후보

---

## 8.9 GPT 요약 및 재필터

GPT mini는 전체 뉴스를 요약하지 않고, 중요 후보 뉴스만 요약한다.

### GPT mini 요약 대상

- important_candidate
- KODEX 200 / KODEX 코스닥150 포함 종목 뉴스
- 보유종목 뉴스
- 관심종목 뉴스
- 알림 설정 종목 뉴스
- duplicate_count >= 2
- source_count >= 2
- 시장/섹터/매크로/정책 뉴스 중 importance_score >= 6

### GPT mini 요약 제외

- exclude_candidate
- 단순 특징주
- 단순 장중 급등락
- 광고/행사/사진 기사
- 미포함 개별 종목 뉴스

요약 길이:

- 1000자 이내

### GPT-5 재필터

GPT-5 계열 모델은 요약된 중요 후보 뉴스에 대해 다음 분류를 수행한다.

- 중요
- 주가 영향 가능
- 불필요

중요 또는 설정 기준을 만족하는 뉴스만 알림 후보가 된다.

## 8.10 알림 발송

알림 발송은 Gmail SMTP를 사용한다.

발송 대상:

- 본인 Gmail 계정

발송 제한:

- 하루 최대 200건
- 1시간 최대 50건

발송 기준:

- 뉴스 알림 기준 충족
- 가격 알림 기준 충족
- 중복 알림이 아닌 경우

실패 처리:

- 알림 발송 실패 이력 저장
- Gmail 제한 또는 오류 발생 시 발송 중단
- 하루 한도 초과 시 1회 한도 초과 알림 발송
- 알림 클릭 시 시스템 내 관련 뉴스/종목/거래 화면으로 이동할 수 있는 링크 포함

## 8.11 거래 및 보유현황 계산

거래 기록을 기준으로 보유현황을 계산한다.

계산 기준:

- 평균단가 방식
- 매수 시 수량과 평균단가 갱신
- 매도 시 실현손익 계산
- 보유 수량이 0이 되면 보유 종료 상태 처리
- 현재가 기준 평가금액 계산
- 평가손익과 평가손익률 계산
